# 项目概述

## 项目背景

福州市局为紧跟“互联网＋政务服务”时代发展步伐，大力推进烟草行业网络平台的建设与管理效能，并积极响应国家烟草专卖局印发《关于加快全国烟草生产经营管理一体化平台建设推进烟草行业数字化转型的意见》。不断深化网络平台在烟草行业的政务公开、主题教育、业务管理等方面发挥着积极作用。同时《意见》指出，启动全国烟草生产经营管理一体化平台建设，以形成覆盖全产业链、全治理链、全生态圈和全生产要素的一体化、智能化平台为总体目标。

在此背景下，福州市局深知现有网络平台的部分技术和设备是非自主可控的，存在一定的不稳定性和安全隐患，同时也面临着日益严格的国内网络安全法规和政策的约束。

因此，福州市局决定对网络平台进行国产化改造，以提升系统的可靠性、安全性和性能，并更好地满足企业业务的发展需求和法规要求。从而进一步提升自身的信息化水平和竞争力，实现企业管理的数字化转型和可持续发展。

## 建设目标

按照卷烟货源供应管理规范要求，以尊重市场、满足需求、改善状态为出发点，按照“公平、公正、公开”的原则，对现行的民主网进行优化和升级，以提升系统的可靠性、安全性和性能，并更好地满足企业业务的发展需求和法规要求。该项目主要针对系统以及数据库等方面进行改造和优化，使其更加适应国内网络环境和法规要求，确保企业网络系统的稳定运行和数据安全。

通过改造，福州烟草公司将进一步提升自身的信息化水平和竞争力，实现企业管理的数字化转型和可持续发展。

## 建设内容

本项目主要建设内容为国产化适配性改造，为确保改造后的办事公开、民主

管理、法规资源、主题教育、精益管理五大功能模块能够正常运行，需进行包括

民主管理网后端国产化部署、民主管理网历史数据迁移、国产化前端适配性改造、

系统架构设计等建设。

1.民主管理网后端国产化部署

完成服务器部署、系统软件环境构建、数据库部署与配置、安全体系与运维

管理等民主管理网后端国产化部署。

2.民主管理网历史数据迁移

完成数据备份与准备、数据清洗与转换、执行数据迁移、数据验证与培训支

持等民主管理网历史数据迁移

3.国产化前端适配性改造

完成前端兼容适配、性能优化、适配性测试与验证等国产化前端适配性改造。

4.系统架构优化

完成应用服务器、国产操作系统、数据库服务器、中间件等系统架构。

# 项目进度

本项目已经进行了需求分析、系统设计阶段，目前处于开发和测试等阶段，开发测试结束即可进行上线部署。

# 项目实施计划

## 项目实施内容

福州民主公开网的主要功能模块有：办事公开、民主管理、法规资源、主题

教育、精益管理五大模块。

其中办事公开包含：公告栏、政务公开、企务公开、规章制度、三重一大、党务公开等栏目；

民主管理包含：在线咨询、意见征集、领导信箱、举报投诉等栏目；

法规资源包含：人事管理、财务管理、行政安全、专卖管理、招标采购、营

销管理、内部监督、各级文件、综合管理、重要文库、重要法规等栏目；

主题教育包含：组织机构、工作部署、新闻动态、调查研究、检视问题、整

改落实、学习资料、先进典型等栏目；

精益管理包含：精益文件、精益动态、图片展示、精益改善、经验分享、精

益简报等栏目。

本次国产化适配性改造将基于上述功能模块稳健展开，为确保在满足所有改

造需求的同时，能够保障这些功能模块的正常运行。需求主要分析如下：

1.民主管理网后端国产化部署

本期项目要求对民主管理网后端国产化部署的服务器部署、系统软件环境构

建、数据库部署与配置、安全体系与运维管理等进行国产化适配和改造，满足民

主网后台的系统环境优化、系统深度定制、性能调优、全面监控等民主网特定业

务的需求。

民主管理网后端国产化部署需求具体如下：



| 项目需求 | 国产化适配性改造需求细项 |

| --- | --- |

| 服务器部署 | 完成服务器的物理部署、操作系统安装、网络配置及系统环境优化，包括安全加固措施 |

| 系统软件环境构建 | 安装配置必要的中间件，如 Web 服务器、应用服务器，以及适配的数据库管理系统，确保各组件间的兼容性和高效协同。对系统软件进行深度定制与优 化，以满足民主管理网的特定业务需求。 |

| 数据库部署 与配置 | 对国产化数据库软件进行部署与配置，包括数据库结构调整、存储引擎选择、性能调优等。 |

| 安全体系与运维管理 | 保持民主管理网的部署位置不变，强化内网服务器区的安全防护体系，达到安全等级保护一级标准。利用现有或升级后的机房设备监控系统，实现对服务器、应用程序及网络环境的全面监控。 |



2.民主管理网历史数据迁移

本期项目要求对民主管理网历史数据迁移的数据备份与准备、数据清洗与转

换、执行数据迁移、数据验证与培训支持等数据需求。

民主管理网历史数据迁移需求具体如下：



| 项目需求 | 国产化适配性改造需求细项 |

| --- | --- |

| 数据备份与准备 | 在历史数据迁移启动前，执行全面的数据备份策略，包括全量备份与增量备份，确保原始数据的完整性与可恢复性。准备数据迁移所需的环境与资 源，包括迁移工具、脚本及必要的硬件支持。 |

| 数据清洗与 转换 | （1）对备份的数据进行深入分析，执行数据清洗操作，去除重复、无效或错误的数据。 (2) 根据新数据库系统的要求，进行数据格式的转换与数据结构的调整，确保兼容性。 |

| 执行数据迁移 | 利用专业的数据迁移工具或自定义脚本，将清洗并转换后的数据从原数据库系统迁移到国产化数据库系统中。监控迁移过程，确保数据传输的稳定与高效，及时处理可能出现的错误或异常情况。 |

| 数据验证与 培训支持 | 在历史数据迁移完成后，进行全面的数据验证工作，包括数据完整性、一致性与准确性的检查，确保迁移结果符合预期。组织针对新数据库系统的 培训活动，提升系统管理员与数据库管理员的专业技能，包括理论讲解、实践操作与问题解答，确保他们能够顺利地进行后续的系统维护与管理工作。 |



3.国产化前端适配性改造

本期项目要求对国产化前端适配性改造的前端兼容适配、性能优化、适配性

测试与验证等需求。

国产化前端适配性改造需求具体如下：



| 项目需求 | 国产化前端适配性改造需求细项 |

| --- | --- |

| 前端兼容适配 | 针对福州烟草民主网的使用需求，选择火狐浏览器和 360 浏览器、奇安信可信浏览器作为主要的适配目标，确保前端代码与这两种浏览器的兼容性。通过分类梳理现有系统，对前端代码进行有针对性的调整，确保在所有选定浏览器上均能正确显示与交互。 |

| 性能优化 | 实施一系列前端性能优化措施。包括减少 HTTP 请求次数（如合并 JS、CSS文件及图片）、避免不必要的重定向、优化 Ajax 请求以利用缓存、延迟加载非关键组件、最小化 iframe 使用、减少 404 错误以及页面静态化等策略，以提升系统的响应速度和用户体验。 |

| 适配性测试 与验证 | （1）多版本浏览器测试：针对每种选定的浏览器，至少选择三个稳定版本进行测试，以确保广泛覆盖可能存在的兼容性问题。 (2) 国产化软硬件环境测试：在真实的国产化软硬件环境中进行适配性测试，以验证前端改造在实际运行环境中的表现。 |
