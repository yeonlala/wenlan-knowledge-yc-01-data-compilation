6.2 输入输出描述

1. 登录（/sys-login/login）

输入设计过程

输入方法：以 JSON 格式传递用户名和密码。

数据校验和检查方法

用户名：非空，长度在 3 - 20 个字符，无非法字符。

密码：非空，长度 6 - 20 个字符。

输出形式

成功：返回包含用户信息（如用户 ID、用户名、角色等）和 token 的 JSON 对象。

失败：返回包含错误信息（如用户名或密码错误）的 JSON 对象。

列表（/sys - user/list）

输入设计过程

输入方法：以 JSON 格式传递查询条件，如用户名、状态等。

数据校验和检查方法：检查查询条件是否合法，如日期格式是否正确。

输出形式：返回包含用户列表（每个用户包含 ID、用户名、角色、状态等信息）的 JSON 对象。

输入设计过程

输入方法：以 JSON 格式传递用户信息，如用户名、密码、角色等。

初始化密码修改（/sys - user/initPasswordModify）

输入设计过程

输入方法：以 JSON 格式传递用户 ID 和新密码。

数据校验和检查方法：检查用户 ID 是否有效，新密码长度是否符合要求。

输出形式：返回操作结果（成功或失败）及相应提示信息的 JSON 对象。

修改密码（/sys - user/modifyPassword）

输入设计过程

输入方法：以 JSON 格式传递原密码、新密码和确认新密码。

数据校验和检查方法：原密码正确，新密码和确认新密码一致，新密码长度符合要求。

输出形式：返回操作结果（成功或失败）及相应提示信息的 JSON 对象。

本方案将严格根据技术与管理要求进行设计。首先应根据本级具体的基本要求设计本级系统的保护环境模型，保护环境按照安全计算环境、安全区域边界、安全通信网络和安全管理中心进行设计，内容涵盖基本要求的 5 个方面。同时结合管理要求信息系统的安全保护等级由业务信息安全性等级和系统服务保证性等级较高者决定，因此，对某一个定级后的信息系统的安全保护的侧重点可以有多种组合。

九、非功能性设计

1.系统集成需求

系统与福州烟草现有办公自动化、财务管理、人事管理等系统深度集成，通过数据接口实现用户信息、业务数据的实时共享，避免数据重复录入，提升工作效率。

2.质量需求

正确性

系统业务逻辑和数据处理准确无误，通过多轮测试验证，确保系统在各种场景下稳定运行。

容错性

具备完善的错误处理机制，对用户输入的非法数据或系统异常进行有效处理，避免系统崩溃。

稳定性

本系统设计要具有高稳定性，系统在正常的访问使用过程中能够稳定、可靠地进行业务数据处理。

效率

由于系统可能需要处理并发的业务请求，所以要求系统设计要尽可能紧凑、高效，尽量加快每次业务请求的响应时间，避免出现用户需要长时间等待系统进行数据处理的情况。

3.性能需求

减少 HTTP 请求次数（如合并 JS、CSS文件及图片）、避免不必要的重定向、优化 Ajax 请求以利用缓存、延迟加载非关键组件、最小化 iframe 使用、减少 404 错误以及页面静态化等策略，以提升系统的响应速度和用户体验。

4.扩展性需求

系统要充分考虑今后业务的发展、运营管理体制的改变等因素，系统设计要具有高度的灵活性和可扩展性，以便于在今后业务情况发展、变化时，可以根据应用需求的改变而方便地扩充和调整系统功能，从而满足新的应用需求。

5.用户管理需求

系统与福州烟草统一身份认证平台对接，实现用户账号的统一管理。用户账号的创建、修改、删除在统一身份认证平台进行，系统根据用户角色和权限分配功能访问权限。
