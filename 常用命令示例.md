## 生成模拟项目（`generate_mock_tobacco_project.py`）

- **批次**：默认 `当前目录/第一批`，否则 `--batch "路径\第一批"`。
- **项目文件夹名**：须 **`客户单位_项目名称_年份_项目状态`**（至少 **4** 段，**倒数第二段**为四位年份）。省略 `--project-name` 时脚本自动生成。
- **覆盖**：`--clean`。

| `--scenario` | 说明 |
|----------------|------|
| `random_sparse` | **默认**。阶段一目录池内随机残缺 |
| `full` | 必交齐全 + 阶段一最小标准目录 |
| `missing_dirs` | 缺若干标准目录（可有脚本默认缺项） |
| `missing_required_partial` | 目录较全，缺部分必交 |
| `bad_names` | 不合规命名 / 风险词 |
| `dirs_only` | 仅 **`01_管理确认` + `02_核心资料`** 及 `02` 下选定叶子的**空文件夹** |
| `empty` | 仅空项目根文件夹 |

**参数：** `--customer` / `--project`（**`--project` 勿含下划线**）· `--omit-dirs` / `--omit-required` · `--ext`（默认 `.docx`）· **`--phase`**：`dirs_only` **省略时默认 `2`**（`02` 下设齐 **11** 个叶子空目录）；其它场景省略时默认 `1`。显式 **`--phase 1`** = 仅阶段一 **7** 个叶子；`full` / `empty` 不用 phase。

**空目录模板：**

```bash
python generate_mock_tobacco_project.py --scenario dirs_only --clean
python generate_mock_tobacco_project.py --scenario dirs_only --phase 1 --clean
python generate_mock_tobacco_project.py --scenario dirs_only --phase 1 --project-name "测试市局_演示_2026_待验收" --clean
python generate_mock_tobacco_project.py --scenario empty --clean
```

**其它示例：**

```bash
python generate_mock_tobacco_project.py --scenario random_sparse --seed 42 --clean --ext .pdf
python generate_mock_tobacco_project.py --seed 42 --clean
python generate_mock_tobacco_project.py --scenario full --project-name "测试市局_演示_2026_待验收" --clean --ext .pdf
python generate_mock_tobacco_project.py --scenario missing_dirs --clean
python generate_mock_tobacco_project.py --scenario bad_names --clean
```

## 清理工作区（`clean_workspace.py`）

```bash
python clean_workspace.py --results              # 仅清空「检查结果」
python clean_workspace.py --mock --yes           # 仅删「第一批」下一级项目目录
python clean_workspace.py --all --yes            # 两项都做
python clean_workspace.py --all --dry-run        # 预览
python clean_workspace.py --base "D:\资料上报" --results
```

同目录有 bat；若无反应可用 CMD：`py -3 clean_workspace.py ...`。
