## 运维可视化（`ops_mvp`）

在**仓库根目录**执行（勿在 `ops_mvp/` 子目录内单独跑 `uvicorn app:app`，模块路径会不一致）：

```bash
uvicorn ops_mvp.app:app --host 127.0.0.1 --port 8765
```

浏览器打开：**http://127.0.0.1:8765**。开发调试可加 **`--reload`**。

默认扫描当前目录下 **`第一批`**，报告写入与其同级的 **`检查结果`**（Excel + HTML）。

```bash

```

## 确认单 JSON 拆清单（`2_export_confirmation_bundle.py`）

从各项目 **`01_管理确认`** 下的《项目资料上报确认单》`.json` 生成入库用分类 JSON，并输出到 **`extract_jsons/<项目文件夹名>/`**（含并列 **`docs_metadata/`** 归集副本）。省略文件参数时自动扫描仓库 **`第一批`** 下各项目。

```bash
python 2_export_confirmation_bundle.py
python 2_export_confirmation_bundle.py "第一批\某单位_某项目_2026_待验收\01_管理确认\项目资料上报确认单_20260508.json"
```

## 生成模拟项目（`generate_mock_tobacco_project.py`）

仅三种 **`--scenario`**（脚本默认 **`full`**）。

- **输出根目录**：默认 **仓库根下的 `mockdata/`**（自动创建）；也可用 `--batch` 指定其它路径（相对路径相对仓库根）。
- **项目文件夹名**：须 **`客户单位_项目名称_年份_项目状态`**（至少 **4** 段，**倒数第二段**为四位年份）。省略 `--project-name` 时脚本自动生成四段式名称。
- **覆盖**：`--clean`。

| `--scenario` | 说明 |
|----------------|------|
| `full` | **`02_核心资料`** 下设齐 **01_～07_** 并写入占位材料；**`01_管理确认`** 下写入与生成器同结构的确认单 **.md / .json** |
| `dirs_only` | **`01_管理确认` + `02_核心资料`**，**`02` 下 01_～07_ 为标准空目录**（可用 `--omit-dirs` 排除个别叶子） |
| `missing_required_partial` | **02** 下 **随机建 1～7 个**标准叶目录（不先建齐）；叶目录名与写入材料的个数均随机；可选 **`--seed`** 复现 |

**参数：** `--customer` / `--project`（**`--project` 勿含下划线**）· `--omit-dirs`（仅 `dirs_only`）· `--ext`（默认 `.docx`）· `--seed`（仅 `missing_required_partial`）。

**示例：**

```bash
python generate_mock_tobacco_project.py --scenario full --clean
python generate_mock_tobacco_project.py --scenario dirs_only --clean
python generate_mock_tobacco_project.py --scenario dirs_only --omit-dirs "06_验收交付,07_项目复盘" --clean
python generate_mock_tobacco_project.py --scenario missing_required_partial --seed 42 --clean
python generate_mock_tobacco_project.py --scenario full --project-name "测试市局_演示_2026_待验收" --clean --ext .pdf
```

## 清理工作区（`clean_workspace.py`）

```bash
python clean_workspace.py --results              # 仅清空「检查结果」
python clean_workspace.py --mock --yes           # 仅删「第一批」下一级项目目录
python clean_workspace.py --all --yes            # 两项都做
python clean_workspace.py --all --dry-run        # 预览
python clean_workspace.py --base "D:\资料上报" --results
```

同目录有 `clean_results.bat`、`clean_mock_projects.bat`；若无反应可用 CMD：`py -3 clean_workspace.py ...`。

## 推荐联调（Mock → 机检）

```bash
python generate_mock_tobacco_project.py --scenario full --clean
python 1_check_tobacco_kb_required_files.py mockdata
```

验收脚本第一个参数为扫描根目录；若资料只在 **`第一批`**，仍传 **`第一批`**。报告在 **`检查结果`**。
