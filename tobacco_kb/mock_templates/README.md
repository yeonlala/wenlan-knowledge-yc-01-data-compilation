# 第一阶段 Mock 范例正文（可编辑）

**`r01.md`～`r07.md`** 与 **`PHASE1_RULES`** 一一对应（改规则 id 请同步文件名）。**《项目资料上报确认单》** 的规范版面与导出请用仓库 **`模板示例/确认单生成器/index.html`**（见 **`模板示例/使用说明.md`**）；本目录若另行放置 **`资料上报确认单.md`**，可用于 Mock 在 **`01_管理确认`** 下生成占位正文（无 `python-docx` 时可能输出 `.md`）。

修改后重新执行 **`generate_mock_tobacco_project.py`** 即生效。

## 编码与格式

- 请使用 **UTF-8** 保存（多数编辑器默认即为 UTF-8）。
- 支持常见 **Markdown**，生成 `.docx` 时会尽量还原为 Word 样式（需安装 `python-docx`，见仓库根目录 `requirements.txt`）：
  - 标题 `#`～`######` → 标题 1～6
  - `-` / `*` 无序列表、`1.` 有序列表
  - `|` 表格
  - `>` 引用
  - 单独一行的 `---` → 分隔线
  - 行内 `**加粗**`
- 也可新增 **`rXX.txt`**（与 `rXX.md` 二选一，优先读取 `.md`）；`.txt` 无 Markdown 语法时按普通段落处理。

## 与生成目录的关系

范例正文**仅保存在本仓库 `tobacco_kb/mock_templates/`**，生成 mock 时脚本会读取对应 **`rXX.md` / `rXX.txt`** 写入按规定命名的 `.docx`/`.pdf`/`.xlsx` 等；**不会在生成的资料目录里再放一份模板 `.md` 文件**。
