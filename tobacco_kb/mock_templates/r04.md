# 业务流程说明（配合流程图使用）

> 流程图文件（Visio、PNG、PDF 等）请放在同一阶段目录；本文用于文字说明 **泳道、关键分支与例外处理**。

## 1. 流程名称

卷烟物流配送智能排班与调度流程

## 2. 参与角色

| 角色 | 职责 |
|------|------|
| 营销系统（上游） | 每日推送零售户订货清单、SKU 规格及实物量数据。 |
| 物流调度员 | 维护车辆/司机基础信息，对自动排班结果进行人工微调及确认。 |
| 智能算法引擎 | 执行 VRP 路径优化算法，生成初始排班方案。 |
| 仓储系统（下游） | 接收已确认的排班指令，启动分拣线与装车位分配。 |

## 3. 步骤说明

1. **开始：** 每日凌晨 02:00，系统自动检测营销系统当日截单后的“订货库”数据。
2. **数据预处理：** 系统自动剔除地址信息异常、订单总量为零的无效记录，并按配送中心（DC）进行分区划归。
3. **算法自动化调度（核心）：** * 调用 VRP 路径优化算法，综合考虑车辆额定载重（体积/重量）、送货时间窗、路网禁行限制。
    * 系统输出“车辆-路线-零售户”的最优组合方案。
4. **人工合规性审批：** * 调度员查看系统生成的路线图，若发现突发路况（如修路、交通管制），可手动执行“订单合并”或“跨线路调整”。
    * **通过**：点击“下发指令”，数据流向仓储 WMS。
    * **退回**：重新设定约束条件，触发算法二次计算。
5. **结束：** 系统生成《装车清单》与《送货路径图》，通过 API 同步至分拣控制系统。

## 4. 例外场景

| 场景 | 处理方式 |
|------|----------|
| **算法无解（超出运力）** | 系统自动触发“运力缺口预警”，提示调度员调拨第三方社会物流车辆，或启用应急预备车队。 |
| **GIS 地图偏移** | 若定位点与实际配送点偏差 > 500m，调度员需手动在地图上“重锚定”，并将位置信息回传至零售户基础数据库。 |
| **网络通信中断** | 平台启用“本地备份模式”，支持调度员上传离线 CSV 格式订货清单进行半自动排班。 |
| **临时订单追加** | 仅限特定级别订单（如战略客户补货），系统支持动态插单，实时刷新受影响路线的剩余载重。 |
